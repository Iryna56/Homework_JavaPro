1.Что такое Set/List/Queue и для чего они нужны? это интерфейсы или классы?
Это интерфейсы. Они расширяют интерфейс Collection. List хранит упорядоченные элементы, Set - коллекции, которые не содержат повторяющихся элементов, Queue - интерфейс для реализации очереди в джава.
2.Что такое ArrayList?
Это модификация массива, который является динамическим, может расширятся.
3.Что гарантирует Set? Что не гарантирует? 
Set гарантирует то, что в коллекции не может попасть повторяющееся значение, но не гарантирует того, что значения будут упорядочены, если их не отсортировать
4.Что не гарантирует List? Что гарантирует?
Гарантирует, что элементы будут упорядочены и есть доступ к индексу
5.Что стоит во главе иерархии Java Collection Framework?
Collection
6.Какой вспомогательный класс есть для работы с массивами?
Arrays, Collections
7.Можно ли написать следующую конструкцию? Collection<int> col = new ArrayList<>();
Нет, примитивы указывать нельзя, только ссылочные типы 
8.В чем разница между двумя подходами?
При 
for (int i = 0; i < array.length; i++) {
    //тут логика
    }

    for (int i : array) {
        //тут логика
    }
Цикл for обычно применяется к статическим массивам, for each - к динамическим. В for each  мы проходимся по массиву от начала до конца, в цикле фор - по указанному отрезку.
