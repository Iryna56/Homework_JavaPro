1)Класс - это шаблон для создания объекта. А объект - это экземпляр класса. 
Класс определяет структуру и поведение, а объект содержит структуру и поведение, которые определены классом.
Стиль кода, который пишется в java - это объект.
2)Packages нужны для инкапсуляции группы классов, подпакетов и интерфейсов. Это контейнер для хранения группы классов.Внутри пакетов некоторые классы могут быть скрыты, а некоторые открыты к использованию. 
Import в java - это оператор, который определяет и сообщает компилятору путь к классу, на который ссылается код.
3)Название класса должно быть именем существительным,и должно быть названо в соответствии с целью, которая должна быть достигнута и с заглавной буквы каждого слова(CamelCase). Названия методов должны быть глаголами в смешанном регистре с первой буквой в нижнем регистре и с заглавной буквой каждого внутреннего слова.Документацию, подтверждающую эти стандарты, можно найти на сайте Oracle в разделе Naming Convections.
4)Конструкторов в классе может быть неограниченное количество, но как минимум один.
5)Если не указать ни одного конструктора в классе, то конструктор создается компилятором по умолчанию.
6)This нужен для предотвращения конфликта имен. К примеру, если переменная экземпляра и переменная конструктора имеет одно название.
7)Инкапсуляция - это обьединение данных и методов работы с этими данными в одной упаковке("капсуле"). Способами достижения инкапсуляции и сокрытия есть установление нужных модификаторов доступа и использование геттеров и сеттеров.
8)Модификаторы доступа:
Public-публичный, поля и методы видны другим классам и из текущего пакета и из внешних пакетов и доступны всей вселенной)
Protected-доступен внутри класса и его наследниках, а также из текущего и внешнего пакетов
Default- или метод по умолчанию имеет доступ всем классам внутри текущего пакета
Private-доступен только внутри класса
9)Геттеры и сеттеры-методы доступа в java. Используются для получения или изменения значения поля класса, если поле имеет модификатор доступа private. Также эти методы еще используются,если нужно внутрь метода положить какую-нибудь логику.
